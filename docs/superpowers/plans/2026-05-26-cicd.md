# CI/CD Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a four-job GitHub Actions pipeline (validate → build → lighthouse → deploy) that quality-gates every push to `main` and auto-deploys optimized files to `gh-pages`.

**Architecture:** Single workflow file with sequential jobs sharing a build artifact. `validate` checks HTML syntax and links; `build` runs sharp image optimization and uploads `_site/` as an artifact; `lighthouse` serves the artifact and asserts all Lighthouse categories ≥0.90; `deploy` pushes the artifact to the `gh-pages` branch.

**Tech Stack:** GitHub Actions, html-validate, lychee-action, sharp (already installed), @lhci/cli, http-server, peaceiris/actions-gh-pages

## Task 1: Image optimization script

**Files:**
- Create: `scripts/optimize-images.js`

- [ ] **Step 1: Write the script**

```javascript
// scripts/optimize-images.js
const sharp = require('sharp');
const fs = require('fs');
const path = require('path');

function getImageFiles(dir) {
  if (!fs.existsSync(dir)) return [];
  const results = [];
  for (const entry of fs.readdirSync(dir, { withFileTypes: true })) {
    const full = path.join(dir, entry.name);
    if (entry.isDirectory()) {
      results.push(...getImageFiles(full));
    } else if (/\.(jpe?g|png)$/i.test(entry.name)) {
      results.push(full);
    }
  }
  return results;
}

async function optimizeImages(imagesDir) {
  const files = getImageFiles(imagesDir);
  console.log(`Optimizing ${files.length} images in ${imagesDir}`);
  for (const file of files) {
    const ext = path.extname(file).toLowerCase();
    const tmp = file + '.tmp';
    if (ext === '.jpg' || ext === '.jpeg') {
      await sharp(file).jpeg({ quality: 85, progressive: true }).toFile(tmp);
    } else {
      await sharp(file).png({ compressionLevel: 9 }).toFile(tmp);
    }
    fs.renameSync(tmp, file);
    console.log(`  optimized: ${path.relative(process.cwd(), file)}`);
  }
  console.log('Done.');
}

const target = process.argv[2] || path.join(process.cwd(), '_site', 'images');
optimizeImages(target).catch(err => { console.error(err); process.exit(1); });
```

- [ ] **Step 2: Test the script locally against the real images directory**

```powershell

```

Expected output:
```

  optimized: images\gallery\gallery-01.jpeg
  ...
Done.
```

If sharp throws `Cannot find module 'sharp'`, run `npm install` first.

- [ ] **Step 3: Verify images are still valid after optimization**

Open `index.html` in a browser (or any image viewer). Gallery images and logo should display normally. If any image is corrupted (blank/broken), the `.tmp` file handling failed — check that `fs.renameSync` completed for all files.

## Task 3: GitHub Actions workflow

**Files:**
- Create: `.github/workflows/deploy.yml`

- [ ] **Step 1: Create the workflows directory**

```powershell
New-Item -ItemType Directory -Force .github/workflows
```

- [ ] **Step 2: Write the workflow**

```yaml
name: CI/CD

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  # ── 1. VALIDATE ──────────────────────────────────────────────────────────────
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Validate HTML
        run: npx --yes html-validate index.html 404.html

      - name: Check internal links (hard fail)
        uses: lycheeverse/lychee-action@v2
        with:
          args: >-
            --verbose --no-progress --timeout 10
            --exclude-mail --exclude-external
            index.html 404.html
          fail: true
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Check external links (warnings only)
        uses: lycheeverse/lychee-action@v2
        with:
          args: >-
            --verbose --no-progress --timeout 10
            --exclude-mail
            index.html 404.html
          fail: false
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

  # ── 2. BUILD ─────────────────────────────────────────────────────────────────
  build:
    needs: validate
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm install

      - name: Copy site to _site
        run: |
          rsync -av \
            --exclude='.git' \
            --exclude='node_modules' \
            --exclude='.github' \
            --exclude='.remember' \
            --exclude='_site' \
            --exclude='scripts' \
            --exclude='docs' \
            --exclude='hooks' \
            --exclude='package.json' \
            --exclude='package-lock.json' \
            --exclude='lighthouserc.json' \
            . _site/

      - name: Optimize images
        run: node scripts/optimize-images.js _site/images

      - uses: actions/upload-artifact@v4
        with:
          name: site
          path: _site/

  # ── 3. LIGHTHOUSE ─────────────────────────────────────────────────────────────
  lighthouse:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Download site artifact
        uses: actions/download-artifact@v4
        with:
          name: site
          path: _site/

      - name: Install Lighthouse CI and http-server
        run: npm install -g @lhci/cli http-server

      - name: Run Lighthouse CI
        run: lhci autorun
        env:
          LHCI_GITHUB_APP_TOKEN: ${{ secrets.GITHUB_TOKEN }}

  # ── 4. DEPLOY ─────────────────────────────────────────────────────────────────
  deploy:
    needs: lighthouse
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main' && github.event_name == 'push'
    steps:
      - name: Download site artifact
        uses: actions/download-artifact@v4
        with:
          name: site
          path: _site/

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./_site
```

```bash

git commit -m "feat: add CI/CD pipeline — validate, build, lighthouse, deploy"
```

## Task 4: Push and verify the pipeline

- [ ] **Step 1: Push to GitHub**

```bash
git push origin main
```

- [ ] **Step 2: Open Actions tab**

Go to `https://github.com/jojo6550/newborn-baby-initiat/actions`. A run named "CI/CD" should appear within seconds.

- [ ] **Step 3: Watch each job pass**

Expected sequence: `validate` → `build` → `lighthouse` → `deploy`. All green. Total time ~3–6 minutes.

- [ ] **Step 4: If `validate` fails with html-validate errors**

Read the error output. Common issues and fixes:

| Error | Fix |
|-------|-----|
| `Missing "alt" attribute` | Add `alt=""` (decorative) or `alt="description"` to the `<img>` |
| `Element "script" not permitted` | Add `type="text/javascript"` or move script to end of body |
| `Unclosed element` | Fix the unclosed tag in the reported file/line |

After fixing, commit and push again.

- [ ] **Step 5: If `validate` fails with lychee broken internal links**

The error output names the broken URL and the file it's in. Fix the href/src in that file. Common cause: a renamed image file whose reference wasn't updated.

- [ ] **Step 6: If `lighthouse` fails with score below 0.90**

The LHCI output names the failing category and score. Common fixes:

| Failing category | Likely cause | Fix |
|-----------------|--------------|-----|
| Performance <0.90 | Large unoptimized images, render-blocking scripts | Add `loading="lazy"` to below-fold images; move `<script>` to end of `<body>` |
| Accessibility <0.90 | Missing alt text, low color contrast | Add `alt` attrs; check contrast with browser DevTools |
| Best Practices <0.90 | HTTP links, deprecated APIs, no HTTPS | Change `http://` hrefs to `https://` |
| SEO <0.90 | Missing meta description, small font size | Verify `<meta name="description">` exists in `<head>` |

After fixing, commit and push again to re-run the pipeline.

- [ ] **Step 7: Verify deploy succeeded**

After all four jobs are green, check `https://jojo6550.github.io/newborn-baby-initiat/`. Page should load. If it shows old content, wait 1–2 minutes for GitHub Pages CDN to refresh.

## Post-Implementation Notes

**Adding external links to lychee ignore list:** If the external link check flags a URL that's legitimately unavailable (e.g., a partner site that blocks bots), create `.lycheeignore` at the repo root and add the URL pattern one per line.

**Branch protection (optional):** In GitHub repo Settings → Branches → Add rule for `main`, enable "Require status checks to pass before merging" and select `validate`, `build`, `lighthouse`. This prevents merging PRs that fail quality gates.
